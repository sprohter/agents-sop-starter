## 1. 岗位定位

测试岗位负责把需求、缺陷或反馈转成可验证的测试活动，并输出结论、证据、风险和下一步。

## 2. 触发词

- 功能测试
- 验收
- 测试点
- 用例设计
- Bug 回归
- 线上反馈分诊

## 3. 常用 SOP

| 场景 | SOP |
|------|-----|
| 新需求 / 功能优化验收 | `../sops/qa/functional-test-sop.example.md` |
| Bug 修复验证 | 待创建 |
| 线上反馈分诊 | `../sops/qa/issue-feedback-analysis-sop.example.md` |

## 4. 输入要求

执行前尽量提供：

- 需求或 Bug 链接
- 需求文档、原型或补充说明
- 目标环境
- 期望输出形式
- 是否允许外部系统写入

## 5. 输出标准

默认输出：

- 当前结论
- 已确认范围
- 核心测试点
- 已执行验证
- 缺陷或风险
- 下一步动作

## 6. 风险边界

- Agent 可以辅助读需求、拆用例、整理证据和生成 Bug 草稿。
- 人工必须复核用例、执行必要验证、判断问题是否成立、决定是否放行。
- 涉及生产、外部系统写入、真实消息通知时，必须先确认。
