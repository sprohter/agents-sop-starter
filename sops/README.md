# sops/

> SOP 目录。这里存放可重复执行、可培训、可检查的标准流程。

## 什么内容适合写成 SOP

- 每周、每月或每个项目都会重复执行的流程
- 多人执行时容易口径不一致的流程
- 有明确输入、步骤、输出和验收标准的流程
- 发生过问题，需要固化避免再次遗漏的流程

## SOP 生命周期

| 状态 | 含义 |
|------|------|
| `draft` | 初稿，允许快速迭代 |
| `active` | 当前推荐执行版本 |
| `deprecated` | 已废弃，保留原因和替代入口 |

## 创建步骤

1. 复制 `../templates/sop.template.md`。
2. 放到 `sops/<domain>/<sop-name>.md`。
3. 补齐触发词、输入、步骤、输出、验收标准。
4. 在 `../routing.md` 增加路由。
5. 在对应 `roles/<role-name>.md` 增加引用。

## SOP 输出标准

每次执行 SOP 后，默认输出：

- 当前结论
- 已执行步骤
- 关键依据
- 风险或未确认项
- 下一步动作

## SOP 不放什么

- 不放敏感数据。
- 不放个人路径。
- 不放完整历史聊天记录。
- 不放与流程无关的临时分析。

稳定事实和口径写入 `knowledge/`，流程治理写入 `governance/`。
