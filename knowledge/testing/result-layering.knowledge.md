# 测试结果分层输出规则

status: active
owner: qa-team
last_updated: 2026-06-05

## 结论

测试结论和问题分析结果都应分层表达，至少拆开“已证实”“待确认”“环境阻断”“风险与下一步”。

## 推荐输出层次

| 层次 | 应写什么 |
|------|----------|
| 已证实 | 已完成验证、已有证据支撑的事实 |
| 待确认 | 证据不足、仍需补采或需要他人确认的部分 |
| 环境阻断 | 因环境、权限、依赖、数据前置导致未完成的部分 |
| 风险与下一步 | 当前影响、建议动作、责任归属 |

## 适用场景

- 功能测试结论
- 问题反馈分析结论
- 回归验证结论
- 对外同步的缺陷处理状态

## 常见误区

- 把阻断项写进“通过”结论里
- 把推测写成已确认事实
- 只给过程，不给当前状态
- 只给异常，不给下一步责任归属
