# knowledge/

> 知识沉淀目录。这里存放稳定事实、术语口径、经验卡和可复用判断规则。

## 内容分层

| 层级 | 用途 | 示例 |
|------|------|------|
| M1 索引 | 帮助快速知道该读哪里 | README、目录索引 |
| M2 事实 | 稳定口径、术语、字段解释 | 术语表、团队约定 |
| M3 流程 | 可复用方法，但未必完整到 SOP | 检查清单、判断规则 |
| M4 归档 | 历史案例摘要 | 复盘结论、经验摘要 |

## 什么时候写入 knowledge/

- 一条经验重复出现两次以上。
- 某个判断口径需要多人一致。
- 某个术语、配置、状态含义容易被误解。
- SOP 执行中发现了稳定补充规则。

## 推荐文件类型

- `glossary.md`：术语表
- `common-rules.md`：通用判断规则
- `<topic>.knowledge.md`：主题知识卡
- `<topic>.decision.md`：决策记录

可复制模板：

- `../templates/knowledge-card.template.md`
- `../templates/decision-record.template.md`

## 写入原则

- 只写稳定信息。
- 先写结论，再写依据。
- 不写敏感数据和真实业务样例。
- 如果只是一次性排查过程，不放这里。
