# 路由维护规则

> `routing.md` 决定 agent 看到请求后先读哪里。

## 什么时候补路由

- 用户多次用同一类说法触发同一流程。
- 新增了岗位卡。
- 新增了 active 状态 SOP。
- 发现 agent 经常读错入口。
- 某个触发词命中多个流程，造成冲突。

## 路由条目格式

复制 `../templates/route-entry.template.md` 作为草稿，再写入 `../routing.md`。

最小字段：

| 字段 | 说明 |
|------|------|
| 触发词 / 场景 | 用户可能怎么说 |
| 入口文件 | 应先读取哪个文件 |
| 备注 | 适用边界或排除条件 |

## 优先级

1. 具体 SOP 高于岗位卡。
2. 岗位卡高于通用治理规则。
3. 明确触发词高于模糊场景。
4. 最新用户请求高于历史上下文。

## 冲突处理

出现冲突时：

1. 记录冲突触发词。
2. 比较两个入口的职责边界。
3. 优先保留更具体的入口。
4. 给另一个入口增加排除条件。
5. 必要时新增一个更上层的分流 SOP。

## 路由回归检查

每次大改路由后，至少用 3 类请求自测：

- 典型请求：应命中目标入口。
- 近似请求：不应误命中。
- 模糊请求：应提示澄清或命中通用入口。
