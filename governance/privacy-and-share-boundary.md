# 分享边界与脱敏规则

> 这份框架默认可以分享，但分享前必须确认没有带出业务资料和敏感信息。

## 默认结论

- 对外分享版只放骨架级、通用流程、通用工具说明和去业务化示例。
- 私人备份版可以更全面，但仍不保存凭证密钥和运行态噪音。
- 未分类内容默认不进入 public 仓，先按 `export-profiles.md` 分级。

## 可以保留

| 类型 | 示例 |
|------|------|
| 通用流程结构 | 输入、步骤、输出、验收标准 |
| 通用角色边界 | 岗位职责、触发词、交付物 |
| 通用表达规则 | 先结论后依据、风险分层 |
| 通用治理规则 | 文件放置、路由维护、沉淀标准 |
| 占位符 | `<workspace-root>`、`<issue-id>`、`<your-token>` |

## 不应保留

| 类型 | 处理方式 |
|------|----------|
| 账号、密码、token、cookie | 删除，用占位符替代 |
| 客户、订单、合同、用户隐私 | 删除或完全匿名化 |
| 真实项目名、系统名、库表名、接口名 | 改为通用描述 |
| 个人本机绝对路径 | 改为 `<workspace-root>` |
| 真实线上故障细节 | 只保留抽象经验，不保留可识别信息 |
| 内部群、文档、工单链接 | 删除或替换为 `<internal-link>` |
| 运行态噪音 | 不同步，例如日志、缓存、临时导出、会话状态 |

## 示例改写方式

| 原内容类型 | 推荐写法 |
|------------|----------|
| 某真实系统后台 | `<business-system>` |
| 某真实工单链接 | `<issue-system-link>` |
| 某真实接口路径 | `<api-endpoint>` |
| 某真实数据库表 | `<table-name>` |
| 某个人目录 | `<user-home>` |

## 外发结论

外发前给出三类结论之一：

- `可分享`：未发现敏感信息。
- `需脱敏后分享`：有少量可替换信息。
- `不建议分享`：包含业务资料或高风险信息，无法安全裁剪。

## 两类绝对不同步内容

| 类型 | 说明 |
|------|------|
| 凭证密钥 | 密码、token、cookie、私钥、证书、连接串、含认证信息的配置 |
| 运行态噪音 | `runtime/`、日志、缓存、临时报告、浏览器状态、会话状态、一次性导出物 |
