# 多 Agent 复核与协作规则

> 用于判断什么时候需要另一个 agent、子任务 worker 或人工 reviewer 参与。

## 1. 目标

多 agent 协作的目标是提升质量、扩大覆盖面和控制风险，不是把最终判断外包。

基本分工：

| 角色 | 职责 |
|------|------|
| 主 agent | 负责路由、拆解、派发、整合、最终判断和用户沟通 |
| reviewer agent | 对方案、规则、证据或风险做独立复核 |
| worker agent | 执行边界清晰的小范围探索、整理或低风险修改 |
| 工具脚本 | 处理可重复、可校验、输入输出稳定的机械任务 |
| 人工确认 | 对外部写入、发布、删除、权限变更等高风险动作做授权 |

主 agent 必须重新审核 reviewer 输出，不能把 reviewer 原文直接当最终答复。

## 2. 触发等级

| 等级 | 场景 | 默认处理 |
|------|------|----------|
| L0 单 agent | 简单问答、低风险文案、小范围只读查询 | 主 agent 直接完成 |
| L1 自检 | 普通分析、局部定位、低风险草稿 | 主 agent 用 checklist 自检 |
| L2 并行探索 | 多个互不依赖的文件、资料或方案分支 | 可派 worker 并行只读探索 |
| L3 独立复核 | 测试方案、验收标准、共用 SOP、公开材料 | 使用 reviewer agent 或人工 reviewer 复核 |
| L4 治理复核 | 架构、权限、安全边界、自动化、共享规则、能力发布 | 默认需要独立复核；复核失败时说明降级 |
| L5 高影响动作 | 生产写入、外部系统写入、删除、发布、通知、权限变更 | 必须人工确认，执行后回读验证 |

跨多个等级时按最高等级处理。

## 3. 适合协作的任务

- 方案会成为长期 SOP、治理规则、模板或公开分享材料。
- 任务范围大，主 agent 容易漏掉分支或边界。
- 需要从执行视角、读者视角或安全视角各看一遍。
- 多个文件可以独立检查，彼此不会互相覆盖。
- 高风险动作前需要第二视角确认风险和回退方式。

## 4. 不适合协作的任务

- 目标尚不清楚，无法给 reviewer 一个明确问题。
- 多个 agent 会同时修改同一文件或同一运行态。
- 用户明确要求不要复核、不要开子任务或只要快速结论。
- 任务下一步立即依赖结果，等待 reviewer 会阻塞主线。
- 高风险动作尚未获得人工确认。

## 5. 协作输入边界

给 reviewer 的输入应当尽量小：

1. 目标：要判断什么。
2. 范围：允许看哪些文件、规则或摘要。
3. 禁止项：不得触碰哪些路径、系统或外部动作。
4. 输出格式：结论、依据、风险、建议。

不要把以下内容复制给 reviewer：

- 密码、token、cookie、私钥、证书、连接串。
- 原始日志、完整会话正文、数据库结果或运行态输出。
- 真实客户、订单、事故现场、内部链接或可识别业务样本。
- 未脱敏的本机路径、群 ID、主机名、IP 或账号信息。

如果 reviewer 与主 agent 是同一可信本机环境，也应优先给目标和边界，让 reviewer 自行读取被允许的真源；不要把大段私有材料塞进 prompt。

## 6. 并行写入规则

- 默认不让多个 agent 同时改同一文件。
- worker 写入前必须限定文件清单、允许动作和验收方式。
- 主 agent 负责最终 diff 复核。
- 涉及 `routing.md`、`governance/`、同步脚本、权限配置或公开分享边界时，按 L4 处理。
- 涉及外部系统写入时，按 L5 处理。

## 7. 失败与降级

协作失败时不要伪装成已完成复核。

| 情况 | 处理 |
|------|------|
| reviewer 不可用 | 主 agent 自检；L4/L5 需要说明缺少独立复核 |
| reviewer 超时 | 缩小范围重试，或降级为人工确认 / 主 agent 自检 |
| reviewer 无法访问上下文 | 补充脱敏事实摘要，或换成本机可读模式 |
| reviewer 结论冲突 | 回到真源证据裁决，必要时二次复核 |

最终答复可简要说明：

```text
复核状态：已完成 / 未完成 / 降级完成
有效方式：reviewer agent / worker / 人工确认 / 主 agent 自检
采纳情况：采纳了哪些建议，未采纳哪些及原因
剩余风险：如有
```

## 8. 沉淀要求

如果同类协作反复出现，应沉淀为：

- `routing.md` 触发词。
- `governance/` 规则。
- `sops/` 可执行流程。
- `templates/` 复核请求或交接模板。
- `tools/` 工具说明。
